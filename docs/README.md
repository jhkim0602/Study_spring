# lect_B Study Docs

스프링 실습 코드와 별개로, 주차별 학습 내용을 누적해서 정리하는 문서 폴더입니다.

## 문서 운영 원칙

- 각 주차는 `weekXX/` 폴더 단위로 관리한다.
- 각 주차 폴더 안에는 아래 3개 파일을 기본으로 둔다.
  - `README.md`: 해당 주차 문서 인덱스
  - `practice.md`: 실습 내용 정리
  - `theory.md`: 이론 수업 정리
- 실습 코드는 `src/`에서 관리하고, 학습 내용은 `docs/`에서 관리한다.
- 시험 공부 시에는 `theory.md`를 우선 보고, 실습 복습 시에는 `practice.md`를 본다.

## 추천 구조

```text
docs/
  README.md
  week02/
    README.md
    practice.md
    theory.md
  week03/
    README.md
    practice.md
    theory.md
  week04/
    README.md
    practice.md
    theory.md
  week05/
    README.md
    practice.md
    theory.md
  week06/
    README.md
    practice.md
    theory.md
```

## 왜 폴더 안의 md 분리가 좋은가

`실습/이론` 하위 폴더를 또 만드는 방식보다,

- 경로 깊이가 얕아서 Eclipse에서 찾기 쉽고
- GitHub에서 바로 열어보기 편하고
- 시험 직전에 주차별로 빠르게 훑기 쉽다

현재 단계에서는 `practice.md`, `theory.md` 두 파일로 나누는 방식이 가장 관리하기 좋다.
