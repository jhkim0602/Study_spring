# Week 03

## 주제

3주차는 **"스프링이 객체를 관리한다"**는 말을 처음 제대로 배우는 주차다.  
여기서부터 `Bean`, `ApplicationContext`, `DI` 같은 스프링 핵심 단어가 본격적으로 등장한다.

## 이 주차가 왜 중요한가

2주차까지는 "웹 요청과 화면"이 중심이었다.  
3주차부터는 "그 요청을 처리하는 자바 객체를 누가 만들고 연결하는가"가 핵심이 된다.

즉 3주차는:

- Spring을 단순 웹 프레임워크가 아니라
- **객체 관리 프레임워크**로 이해하기 시작하는 지점이다

## 현재 통합 프로젝트에서 어떻게 읽어야 하는가

현재 `Lect8Application`의 전역 컴포넌트 스캔은 `week04~week07` 중심으로 잡혀 있다.  
그래서 `week03` 패키지는 현재 통합 앱 안에서는 "초기 Bean 개념을 분리해서 읽는 학습용 패키지" 관점으로 보는 것이 맞다.

이렇게 둔 이유:

- 이후 주차와 `AppConfig`, `configSms` 같은 이름이 겹칠 수 있고
- 전부 동시에 스캔하면 충돌 가능성이 있기 때문이다

즉 3주차는 기능 누락이 아니라,  
**기초 개념을 독립적으로 이해시키기 위한 분리 영역**이라고 보면 된다.

## 빠른 요약

- 빈은 스프링이 관리하는 객체다.
- 스프링 컨테이너는 객체를 만들고 연결한다.
- DI는 필요한 객체를 외부에서 주입받는 방식이다.
- 3주차는 이후 모든 주차의 기반이 되는 핵심 주차다.
